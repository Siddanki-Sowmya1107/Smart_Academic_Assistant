# Agentic Smart Academic Assistant

An AI-powered academic assistant built to help university students get faster guidance on academic questions, resources, and task-related support.

## Problem

Students often need to search across multiple university pages, documents, portals, or offices to find simple academic information. This creates delays, confusion, and repetitive manual lookup.

## What We Built

We built a Streamlit-based AI assistant that uses structured prompts and an agent-style workflow to understand a student request, route it to the right type of response, and generate clear guidance.

The assistant supports:
- Academic question answering
- Resource guidance
- Task-based student support
- Structured AI-generated responses
- Edge-case testing for unclear or incomplete questions

## Tools and Technologies

- Python
- Streamlit
- OpenAI API
- Prompt Engineering
- Agent-style workflow logic
- Git and GitHub

## How It Works

1. The user enters an academic question or support request.
2. The system classifies the type of request.
3. A structured prompt is created based on the request type.
4. The AI model generates a clear, student-friendly response.
5. Outputs are reviewed and refined for accuracy, clarity, and usefulness.

## What I Learned

This project taught me that building useful AI tools is not just about calling an API. The real work is understanding the user problem, designing the workflow, testing failures, improving the output, and making the tool easy for people to use.

I also learned how important prompt structure, response validation, and edge-case testing are when building AI applications for real users.
